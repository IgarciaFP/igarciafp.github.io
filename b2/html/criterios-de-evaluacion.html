Criterios de evaluación

En esta tabla se relacionan las actividades del proyecto con los criterios de evaluación, el tipo de agrupamiento, la duración estimada y las herramientas recomendadas.

Importante: Cada actividad contribuye al desarrollo de competencias específicas del módulo.
Actividad Criterio de evaluación Agrupamiento Tiempo Herramientas
Identificación de IDEs y herramientas IA a) Identificación de herramientas para diseño y prueba Individual 2 h ChatGPT, VS Code, Copilot
Práctica con Flexbox y Grid b) Creación de componentes visuales Individual 2 h Flexbox Froggy, Grid Garden
Desarrollo del front-end c) Definición de propiedades y valores Pareja 4 h VS Code + Copilot
Eventos dinámicos d) Asociación de eventos y acciones Pareja 4 h VS Code
Pruebas del sistema e) Pruebas unitarias Pareja 2 h Claude, testing tools
Documentación f) Documentación del desarrollo Pareja 2 h ChatGPT
Componentes reutilizables g) Empaquetado de componentes Pareja 2 h GitHub
Integración final h) Desarrollo de aplicación completa Pareja 2 h VS Code + servidor web

Resolución de problemas técnicos

Durante el desarrollo del proyecto pueden surgir dificultades técnicas. Para afrontarlas:

  • El profesorado anticipará los problemas más comunes
  • Se propondrán soluciones alternativas (plan B)
  • El alumnado deberá investigar y resolver problemas de forma autónoma
Competencia clave: Saber buscar soluciones es tan importante como saber programar.
