Actividad 1: Debate técnico – elección de herramientas para desarrollo frontend

En esta actividad analizarás y compararás distintas herramientas utilizadas en el desarrollo frontend con el objetivo de seleccionar el entorno de trabajo más adecuado para el proyecto.

Contexto: En sesiones previas, cada grupo ha investigado y probado diferentes herramientas de desarrollo, incluyendo editores de código, asistentes de inteligencia artificial, herramientas de testing y sistemas de análisis de código.

El objetivo ahora es contrastar esa información y tomar decisiones fundamentadas desde un punto de vista técnico y profesional.

Aspectos a decidir

Durante el debate se seleccionarán herramientas para distintas fases del desarrollo frontend:

1. IDE o editor de código

  • VS Code + GitHub Copilot
  • Cursor
  • Antigravity
  • Google AI Studio

Se valorará:

  • Rendimiento
  • Facilidad de uso
  • Disponibilidad de extensiones
  • Integración con herramientas de IA

2. Asistentes de IA para programación

  • Google Stitch (prototipado)
  • Jules (github)
  • v0.dev (Vercel)

Se analizará:

  • Calidad de las sugerencias
  • Integración con el entorno de desarrollo
  • Utilidad real en tareas de programación

3. IA para depuración y explicación

  • ChatGPT
  • Claude Code

Se valorará:

  • Capacidad para explicar código
  • Detección de errores
  • Propuestas de mejora

4. Testing y análisis de código

  • ESLint
  • Prettier
  • Playwright
  • Cypress

Se analizará:

  • Mantenimiento del estilo del código
  • Automatización de pruebas
  • Facilidad de integración

Desarrollo del debate

Fase 1: Presentación

Cada grupo dispondrá de 3–4 minutos para presentar una herramienta:

  • Qué es
  • Para qué sirve
  • Funciones principales
  • Ventajas
  • Inconvenientes
  • Breve demostración (si es posible)

Fase 2: Preguntas

El resto del alumnado podrá plantear preguntas relacionadas con:

  • Integración con otras herramientas
  • Rendimiento
  • Facilidad de uso
  • Documentación disponible
  • Limitaciones

Fase 3: Comparación

Se realizará un análisis conjunto de las herramientas según los siguientes criterios:

  • Productividad
  • Facilidad de aprendizaje
  • Funciones de inteligencia artificial
  • Estabilidad
  • Adecuación al tipo de proyecto
Importante: Las opiniones deben basarse en pruebas reales y argumentos técnicos, no en preferencias personales.

Fase 4: Decisión

  • Se realizará una votación argumentada
  • Se seleccionarán las herramientas más adecuadas

Fase 5: Resultado final

Se elaborará un documento de consenso que incluirá:

  • IDE principal
  • Herramientas de IA
  • Herramientas de testing
  • Otras herramientas recomendadas
Resultado final: Este documento definirá el entorno de desarrollo que se utilizará durante todo el proyecto.

Competencias que se desarrollan

  • Análisis crítico de herramientas tecnológicas
  • Toma de decisiones técnicas
  • Trabajo colaborativo
  • Argumentación y comunicación
